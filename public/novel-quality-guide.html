🚀 系统概览

这是一个基于多 Agent 协作的 AI 小说创作系统。核心思路是让每个 Agent 只干一件事，把创作流程拆解为专业分工：

  • 策划（Planner）：根据主题生成大纲
  • 作者（Writer）：按大纲撰写章节初稿
  • 编辑（Editor）：按评审维度审查并给出修改意见
  • 去AI化（Humanizer）：消除 AI 腔调，增加人味
  • 校编（Proofreader）：修正语病、错别字、标点
  • 读者（Reader）：从读者视角给出可读性评分和反馈
  • 摘要（Summarizer）：生成章节摘要，为后续章节提供上下文
💡 你可以在「设置 → 模型配置」中为每个角色独立配置模型和温度，甚至让多个作者模型轮换写作，实现风格交替。

⚡ 快速开始：3步写小说

Step 1：写下你的想法

在「开始创作」页面输入小说主题。没有灵感？点击下方的灵感标签（退婚逆袭、系统觉醒、重生复仇等）快速填充。

Step 2：选择风格

  • 发布平台：番茄、起点、晋江、七猫、飞卢等 10 个平台，每个都有独特的读者群体和文风偏好
  • 作者风格：热血磅礴、少年逆袭、体系严谨、深情宿命等 80 种风格，决定小说的整体调性
  • 生成模式：快速模式（1-2分钟）或精品模式（5-15分钟，多轮精修）
  • 写作风格：工业风（严格量产）或自由风（创意探索）

Step 3：开始创作

点击「开始创作」按钮，系统会自动完成：大纲生成 → 卷规划 → 章节写作 → 多轮审修 → 读者反馈 → 摘要生成。你可以在页面上实时看到每一步的进度和使用的模型。

💡 创作完成后，进入「我的作品」可以查看作品详情、续写下一章、管理创作辅助数据，或导出完整小说文本。

💡 你可以在「设置 → 写作风格」中设置系统默认模式。创建作品时也可以单独覆盖，让不同作品使用不同模式。

⚙️ 系统设置详解

设置页共 7 个 Tab，每个都影响创作行为的某个方面。

写作技能 & 评审维度

写作技能会注入到作家 Prompt 中，作为创作规范。系统预设三套：

  • 8 维精简版：钩子、爽点、人设、情绪、画面、节奏、语言
  • 15 维标准版：增加期待感、代入感、情节价值、逻辑自洽、对话质量、创新度
  • 33 维完整版：涵盖开篇结构、人物塑造、情节叙事、情绪节奏、文笔语言、世界观、商业价值七大板块

评审维度用于编辑审稿阶段。编辑会逐一检查每个维度是否达标，给出「通过/不通过」结论。维度数量越多，审稿越严格。

💡 你可以在设置页直接编辑 skill 文本和维度列表。切换预设时，系统会自动保存当前编辑内容到对应预设槽位，三套数据互不干扰。

章节字数配置

控制每章的目标字数和容忍区间：

  • targetWords：目标字数（默认 2000）
  • minWords / maxWords：常规容忍区间（默认 1800-2200）
  • absoluteMin / absoluteMax：绝对硬边界（默认 1600-2500）

这些配置同时影响：

  1. Prompt 模板中的字数要求变量（{{targetWords}} 等）
  2. Fitness 评估中的字数得分计算
  3. 导入作品时的章节切分策略

模型配置

这是整个系统最核心的设置，决定每个 Agent 用什么模型、什么温度生成内容。

Provider 配置

系统支持 5 种 Provider：

Provider类型说明
yuanbao原生 API腾讯元宝，直接调用 API
doubao浏览器自动化豆包，需要打开豆包网页并登录
qwen浏览器自动化通义千问，需要打开千问网页并登录
kimi浏览器自动化Kimi，需要打开 Kimi 网页并登录
openai兼容 APIOpenAI / 百炼 / MiniMax / DeepSeek 等兼容接口

每个 Provider 需配置：别名、BaseURL、API Key、可用模型列表。

作者轮换模型 ⭐

启用后，系统会按章节号循环切换 writer 模型：

  • 第 1 章 → 列表第 1 个模型
  • 第 2 章 → 列表第 2 个模型
  • 第 N 章 → (N-1) % 模型数量

轮换只影响 writer（作者）角色，包括初稿写作和改稿修订。同一章内的初稿和改稿使用同一模型，保证风格一致。

默认配置（未启用）：

  • 作家A：yuanbao / deepseek-v3 / T=0.85
  • 作家B：yuanbao / deepseek-r1 / T=0.85
角色默认模型与温度

19 个角色，每个可独立配置 Provider + 模型 + 温度：

角色用途建议温度

编辑（改稿）editor 审阅修改0.8~0.9

🗺️ 创作辅助：世界观、人物、剧情线、地图

在「我的作品」中选择一个作品，进入详情页，切换到「创作辅助」Tab，可以使用以下功能：

世界观记忆库

记录作品的设定信息：力量体系、势力分布、历史背景、特殊规则等。支持分类、标签、重要性标记。写作时系统会自动将相关内容注入 Prompt。

人物设定

创建角色卡片：姓名、别名、角色类型（主角/反派/配角）、状态、外貌、性格、目标、背景故事。支持人物关系图谱，可视化展示角色间的关联。

剧情线图谱

为每条剧情线（主线/支线/暗线）创建时间轴节点，标注每章发生的关键事件。支持颜色标记和展开/折叠，方便追踪多线叙事。

地图图谱

构建作品的世界地图：区域划分、层级关系、区域间的连接（道路/传送阵/海域等）和旅行时间。适合世界观宏大的作品。

💡 创作辅助数据自动注入到所有 outline 和 writing Prompt 中。你不需要手动复制粘贴，系统会在写作时自动提取相关内容。数据越多，AI 对作品世界的理解越深入。

📐 套路模板库

系统内置 5 种经典网文套路模板：退婚流、系统流、种田流、凡人流、签到流。每种模板包含：

  • 节拍结构：从开局到完结的阶段性目标，每阶段预计章节数
  • 代表作：参考作品，帮助理解套路特征
  • 标签：方便检索和分类

你可以在「套路模板库」页面创建自定义模板，也可以在作品详情页将模板关联到具体作品。关联后，模板内容会自动注入写作 Prompt。

🛠️ 工具页面

Agent 聊天

一个轻量化的 AI 对话页面，支持：

  • 选择任意 Provider 和模型
  • 加载任意 Prompt 模板作为系统提示
  • 多轮对话，支持 SSE 流式输出
  • 温度调节（0~2）

适合快速测试 Prompt、调试模型输出、或进行与创作相关的自由对话。对话内容不保存，刷新即丢失。

Prompt 自动进化

基于历史作品的 Fitness 数据，自动诊断 Prompt 缺陷并生成改进变体。系统会：

  1. 分析低 Fitness 章节的共同问题
  2. 生成多个 Prompt 改进版本
  3. 用历史数据预测每个版本的 Fitness 提升幅度
  4. 推荐是否应用改进版本

✅ 如何验证功能是否生效

验证写作技能是否生效

  1. 进入「设置 → 技能」，在 skill 文本开头加一句明显的话，如 【测试标记】这是自定义技能
  2. 保存设置
  3. 开始创作一章
  4. 查看 works/<workId>/chapter_X_prompt.txt，搜索「测试标记」——出现即生效

验证评审维度是否生效

  1. 切换评审预设（8/15/33 维）
  2. 写一章
  3. 查看 works/<workId>/chapter_X_edit_prompt.txt
  4. 确认编辑 Prompt 中的维度清单数量与预设一致

验证模型轮换是否生效

  1. 进入「设置 → 模型配置」，启用「作者轮换模型」
  2. 配置至少 2 个不同的轮换模型（如 deepseek-v3 和 deepseek-r1）
  3. 保存设置
  4. 连续写两章
  5. 观察前端步骤进度条中的模型名：第 1 章和第 2 章应该显示不同的模型
  6. 或检查数据库：SELECT number, models FROM chapters WHERE workId = '...'

验证创作辅助是否注入

  1. 在「我的作品 → 创作辅助」中添加几条世界观设定和人物
  2. 续写下一章
  3. 查看 works/<workId>/chapter_X_prompt.txt
  4. 确认 Prompt 末尾包含「世界观上下文」区块，且内容正确

🎯 附录：什么样的小说才是好小说？

以下是六维视角的解读，供创作时参考：

📐 编辑视角：结构、节奏与市场

黄金三章、人物行动驱动情节、节奏如心电图、伏笔闭环、符合平台调性。

✍️ 作者视角：表达、共情与生命力

真诚比技巧重要、独特的声音、让角色自己活下去、细节即真实、写你想读的书。

📖 读者视角：沉浸、爽感与陪伴

沉浸感第一、主角值得被喜欢、情绪价值到位、期待感是追更动力、成为精神陪伴。

📊 平台算法视角：数据、留存与分发

完读率第一、追更率决定天花板、标签匹配、互动数据放大、适配滑动习惯。

📚 出版人视角：IP 价值与全版权运营

世界观可延展、角色符号化、情感母题普适、视觉化场景丰富、长生命周期。

🎬 编剧视角：冲突、视觉与改编友好度

每章强冲突、场景集中、高光可截图、情绪曲线清晰、金句与名场面。

好小说 = 扎实的情节 × 真诚的情感 × 难忘的陪伴感 × 算法友好 × 改编潜力 × 长尾价值
